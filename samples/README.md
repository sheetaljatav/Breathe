# Sample data

Fabricated input files used for end-to-end testing. Each file is shaped to match what the real source actually produces. The research behind those shapes is in `SOURCES.md`.

| File | Source type | What it exercises |
|---|---|---|
| `sap_se16n_export_2026Q1.csv` | SAP | 48 rows. German decimals and dates, unit conversion (`L`, `ST`, plus `GAL` as a deliberate data-entry mistake), four plant codes including one unmapped (`ZZZZ`), one anomalous quantity (15,450 L versus a ~1,200 L median), one invalid date (`33.03.2026`), procurement SKUs (`OFFICE_*`, `IT_*`, `SVC_*`) routing to Scope 3. |
| `utility_portal_export_meter_xyz.csv` | Utility (CSV) | Billing periods that span calendar months, a unit inconsistency (one row in MWh while the rest are kWh), one anomaly (1,075,000 kWh against a ~92,000 median). |
| `utility_bill_acme_facility_03_2026.pdf` | Utility (PDF) | Text-extractable PDF bill, pdfplumber path. Generated by `python samples/_generate_utility_pdf.py` (the binary isn't committed). |
| `concur_reporting_v4_trip_response.json` | Travel | Concur Reporting API v4 shape. Three trips, mixed AIR / LODGING / CAR segments. AIR with `distance_km: null` (forces airport lookup) and one unresolvable airport (`XYZ`). |

## Why these shapes are realistic

`SOURCES.md` has the full per-source research. In short:

- **SAP SE16N CSV** is the format an analyst gets from the SAP BASIS team when they ask for "the MM data". Semicolon-delimited, UTF-8 with BOM, German decimal commas, German column headers (`Werk`, `Buchungsdatum`).
- **Utility portal CSV** matches the export schema commercial portals use: `meter_id`, billing-period start and end, kWh or MWh consumption, rate class, demand kW. Time-of-use breakdowns appear when the rate class supports them.
- **Utility PDF bill** models a Con Edison-style commercial bill: account, service address, meter ID, billing period, consumption table, charges. Real utility PDFs vary; production needs a template per utility.
- **Concur Reporting v4 JSON** is the response shape from `/reportingapi/v4.0/reports/{id}/expenses`. Trips contain typed segments with vendor codes and (sometimes) distances.
